The Reading Room design language: aged paper, warm ink, oxblood leather and antique gold. Quiet, literary, unhurried — a private library in your pocket, not another feed. These mockups are the design source of truth for the Flutter app (v1 slice).

4g · Public Kitabi profile The same profile screen every "view Kitabi profile" door opens onto — reached from an author page (4f), a search result, or a lending connection — not a bespoke author-only page. Header matches your own profile (12) exactly; what's new is the Works tab sitting next to Library in the same segmented control the lending ledger already uses for Lent out/Borrowed (8b). Library can be locked (🔒, tap does nothing) while Works stays open — a private reader can still be a public author, independent toggles on purpose.

Area 3

Library — the real bookshelf

Covers-first, always (the #1 gripe with Libib). Personal shelves are chips, a book’s page is split into the book (shared catalog) and your copy (private), and adding a book is a camera-first flow.

Cover treatments. Every cover — generated or real — sits in the same frame: rounded corners, left spine shade, drop shadow. With no image, Kitabi typesets a quiet cover from title + author on a colour derived from the book. When the user uploads a photo (or the catalog has one), the image fills the same frame edge-to-edge, and favourites ribbons or lending bands overlay it identically — so a shelf of mixed real and generated covers still reads as one bookshelf.

Area 4

Lending — the ledger, both ways

The wedge feature, styled as what it is: a ledger. Each lending is a record — who, when, due — with return as a satisfying stamp. Free and first-class (Libib paywalls this). It runs both directions: what you’ve lent out, and what you’re borrowing — self-logged, or linked automatically when the friend who lent it to you is a Kitabi user too.
